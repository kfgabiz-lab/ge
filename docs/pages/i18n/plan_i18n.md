## 1. 개요 및 비즈니스 맥락

### Problem Statement
현재 시스템에 다국어(i18n) 관리 체계가 없어, 향후 한국어·영어 다중 언어 서비스 확장 시 텍스트 관리가 불가능한 상태다.

### Business Objective
관리자가 화면에서 직접 번역 키(key)와 각 언어별 텍스트를 등록·수정·삭제할 수 있는 다국어 관리 체계를 구축한다.

### Success Metric
- 다국어 항목 CRUD 기능 정상 동작
- key 유니크 중복 방지 처리
- 검색(key·한국어·영어)으로 원하는 항목 즉시 조회 가능

### 메뉴 위치
```
설정(Settings)
  └── 다국어 관리 (이미 생성 완료)
```

|------|------|------|
| 다국어 목록 | /admin/settings/i18n | 목록 조회 + 검색 + 등록 버튼 |

## 3. 화면 상세 명세 및 핵심 기능

### 3-1. 다국어 목록 화면

**레이아웃**
- 상단: 페이지 헤더 (제목 "다국어 관리" + 설명 문구 + "항목 추가" 버튼)
- 중단: 검색 영역 — 입력 필드 3개 (key / 한국어 / 영어) + 검색 버튼 + 초기화 버튼
- 하단: 테이블 목록

**테이블 컬럼 구성**
| 컬럼 | 설명 | 정렬 |
|------|------|------|
| No | 순번 | 가운데 |
| Key | 번역 키 (예: BTN.SAVE) | 왼쪽 |
| 한국어 | 한국어 번역 텍스트 | 왼쪽 |
| 영어 | 영어 번역 텍스트 | 왼쪽 |
| 등록일 | 항목 등록 일시 | 가운데 |
| 관리 | 수정·삭제 버튼 | 가운데 |

**페이지네이션:** 1페이지 20건, 이전/다음 방식

---

### 3-2. 등록 / 수정 Drawer

**공통 구조**
- 우측 슬라이드인 (권한관리와 동일한 420px 폭)
- 헤더: 제목("항목 추가" / "항목 수정") + 닫기 버튼
- 바디: 입력 폼
- 푸터: 취소 버튼 + 저장/등록 버튼

**폼 필드**
| 필드 | 필수 | 설명 |
|------|------|------|
| Key | 필수 | 영문·숫자·점(.)만 허용, 수정 시 변경 불가(잠금 표시) |
| 한국어 | 필수 | 한국어 번역 텍스트 |
| 영어 | 선택 | 영어 번역 텍스트 (미입력 허용) |

---

### 3-3. 삭제 Confirm 모달
- 내용: `'[key명]' 항목을 삭제하시겠습니까?`
- 버튼: 취소 / 삭제하기(빨간색)

---

## 4. 혁신 및 개선 제안

- **추후 확장성 고려:** 현재 한국어·영어 2개 언어만 지원하지만, 컬럼 구조를 추후 언어 추가 시 유연하게 확장할 수 있도록 설계 (현재는 DB 컬럼 방식으로 단순 구현)
- **key 자동완성:** 추후 등록된 key 목록을 빌더 필드 라벨에 연동하는 기반 데이터로 활용 예정

## 5. 비즈니스 로직 및 예외 정책

| 규칙 | 내용 |
|------|------|
| key 유니크 | 동일 key 중복 등록 불가 → 409 에러 반환, "이미 사용 중인 Key입니다" 토스트 표시 |
| key 형식 | 영문(대소문자)·숫자·점(.)만 허용, 공백/특수문자 불가 → 입력 시 즉시 유효성 검사 |
| key 수정 불가 | 등록 후 key 변경 불가 (수정 Drawer에서 잠금 처리) |
| 한국어 필수 | 한국어 미입력 시 저장 불가 |
| 영어 선택 | 영어 미입력 시 빈 문자열로 저장 허용 |
| 검색 조건 | key·한국어·영어 각각 독립 입력, 입력된 조건만 AND 조합으로 검색 |
| 삭제 확인 | 삭제 전 반드시 Confirm 모달 표시 |
