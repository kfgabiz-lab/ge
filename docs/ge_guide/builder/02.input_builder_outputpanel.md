## 0. 핵심 아키텍처 원칙

### 0-1. 빌더 3개, 결과는 하나

| 빌더 경로 | 용도 | templateType |
|---|---|---|
| `/admin/templates/make/widget` | 위젯 자유 조합 | **`PAGE`** |
| `/admin/templates/make/list` | 검색+목록 코드 생성 | **`LIST`** |
| `/admin/templates/make/grid-layout` | 격자 레이아웃 목록 | **`PAGE`** |

> **어느 빌더에서 만든 `PAGE` 타입 템플릿이든, 운영 페이지는 동일한 경로로 렌더링된다.**

### 0-2. 모든 PAGE 템플릿은 하나의 경로로 렌더링

```
/admin/widgetSub/{slug}
파일: bo/src/app/admin/widgetSub/[slug]/page.tsx
```

`templateType` 분기가 없다. DB에서 불러온 `configJson`의 `widgetItems` 배열을 그대로 `PageGridRenderer`에 전달할 뿐이다.

### 0-3. configJson 구조는 단일 형식

```json
{
    "widgetItems": [
        {
            "id": "wi-1",
            "colSpan": 12,
            "rowSpan": 6,
            "contents": [
                { "id": "ci-1", "colSpan": 12, "rowSpan": 5, "widget": { "type": "form", ... } }
            ]
        }
    ]
}
```

> `fieldRows` / `tableColumns` 등 다른 구조는 **LIST 타입** 전용. PAGE 타입에서는 절대 사용하지 않는다.

### 0-4. templateType 분기 절대 금지

```typescript
// ❌ 절대 금지
if (templateType === 'QUICK_LIST') { ... }
if (templateType === 'QUICK_DETAIL') { ... }

// ✅ 올바른 방법
const raw = JSON.parse(res.data.configJson);
const items: PageWidgetItem[] = raw.widgetItems ?? [];
```

## 1. 개요

빌더 우측 출력 패널은 단순한 미리보기가 아니다.  
**빌더 템플릿 · 빌더 미리보기 · 운영 페이지 — 세 곳에서 완전히 동일한 UI가 나와야 한다.**

```
┌──────────────────────────────────────────────────────────────────┐
│  진입점 A                진입점 B              진입점 C            │
│  builder-contents-layout 빌더 미리보기         운영 페이지         │
│                          (make/*/page.tsx)     (widgetSub/[slug]) │
│       ↓                        ↓                    ↓            │
│  ┌────────────────────────────────────────────────────────────┐  │
│  │  동일한 컴포넌트 스택                                         │  │
│  │  PageLayout → PageGridRenderer → WidgetRenderer →           │  │
│  │  RendererContainer → FieldRenderer                          │  │
│  └────────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘
```

> **핵심 원칙**: UI는 각 진입점에 하드코딩하지 않는다. 컴포넌트에만 존재하고, 세 진입점은 **조립(assemble)** 만 한다.

## 2. 렌더링 계층 구조 ★ 가장 중요

전체 계층은 5단계. 각 레이어는 독립적인 단일 책임을 갖는다.

```
PageLayout
│  역할: 12컬럼 격자 컨테이너, 그리드 가이드라인 on/off
│  파일: bo/src/components/layout/PageLayout.tsx
│
└── PageGridRenderer
│     역할: widgetItems 배열을 받아 각 위젯을 GridCell에 자동 배치
│     파일: make/_shared/components/renderer/PageGridRenderer.tsx
│
└── GridCell (colSpan × rowSpan)
│     역할: 해당 위젯이 차지하는 격자 셀, overflow:clip
│     파일: bo/src/components/layout/GridCell.tsx
│     높이 공식: rowSpan × ROW_HEIGHT - GAP_SIZE
│
└── WidgetRenderer
│     역할: widget.type에 따라 올바른 Renderer 선택
│     파일: make/_shared/components/renderer/WidgetRenderer.tsx
│
└── RendererContainer ★
│     역할: 모든 Renderer 공통 최상위 컨테이너
│           테두리·배경·내부 grid를 하나로 통일
│     파일: make/_shared/components/renderer/RendererContainer.tsx
│
└── FieldRenderer / 컨텐츠 컴포넌트
      역할: 실제 필드(입력창·셀렉트·버튼 등) 렌더링
      파일: make/_shared/components/renderer/fields/
```

## 3. 그리드 수치 기준 (절대 하드코딩 금지)

모든 높이 계산의 단일 출처(Single Source of Truth)는 `GridCell.tsx` 상수다.

```typescript
// bo/src/components/layout/GridCell.tsx
export const ROW_HEIGHT = 80;  // 행 1개의 총 높이 (track + gap 합산)
export const GAP_SIZE   = 8;   // 행 간 gap
```

| 위치 | gridAutoRows | rowGap |
|---|---|---|
| PageLayout | `ROW_HEIGHT - GAP_SIZE` = **72px** | `GAP_SIZE` = **8px** |
| PageGridRenderer | `ROW_HEIGHT - GAP_SIZE` = **72px** | `GAP_SIZE` = **8px** |
| RendererContainer | `ROW_HEIGHT - GAP_SIZE` = **72px** | `GAP_SIZE` = **8px** |

```
rowSpan=1 → GridCell 높이 = 1×80 - 8 = 72px
rowSpan=2 → GridCell 높이 = 2×80 - 8 = 152px
rowSpan=3 → GridCell 높이 = 3×80 - 8 = 232px
```

> ⚠️ **절대 금지**: `80px`, `72px`, `8px` 직접 입력.  
> 반드시 `ROW_HEIGHT`, `GAP_SIZE`, `ROW_HEIGHT - GAP_SIZE`를 import해서 사용한다.

## 4. 각 계층 컴포넌트 상세

### 4-1. PageLayout

12컬럼 격자를 만드는 **페이지 컨테이너**.

```tsx
<PageLayout mode="preview">  {/* 빌더 미리보기·템플릿 */}
    {children}
</PageLayout>

<PageLayout mode="live">     {/* 운영 페이지 */}
    {children}
</PageLayout>
```

| mode | showGrid 기본값 | 설명 |
|---|---|---|
| `preview` | `true` | 격자선·테두리·배경 표시 (g 키로 토글) |
| `live` | `false` | 격자선 없음, 순수 UI만 표시 |

---

### 4-2. GridCell

```tsx
<GridCell colSpan={12} rowSpan={2}>
    <WidgetRenderer mode="preview" widget={widget} contentColSpan={12} />
</GridCell>
```

```typescript
// GridCell Props
interface GridCellProps {
    colSpan: number;       // 1~12
    rowSpan: number;
    children: React.ReactNode;
    className?: string;
    onClick?: () => void;
    colStart?: number;     // 절대 위치 (1~12)
}

// 내부 CSS 계산 (직접 수정 금지)
gridColumn: colStart ? `${colStart} / span ${colSpan}` : `span ${colSpan}`
height: `${rowSpan * ROW_HEIGHT - GAP_SIZE}px`
overflow: 'clip'
```

> `overflow: 'clip'` 이므로 GridCell 높이를 초과하는 컨텐츠는 잘린다. RendererContainer 내부 grid 높이가 GridCell을 초과하지 않도록 주의.

### 4-3. PageGridRenderer ★

위젯 배열 `widgetItems`를 받아 각 위젯을 **GridCell에 자동 배치**하는 렌더러.

#### 두 계층 타입 구조

```typescript
interface PageWidgetItem {
    id: string;
    colSpan: number;
    rowSpan: number;
    contents: PageContentItem[];   // 1 외부 셀 안에 복수 컨텐츠 가능
}

interface PageContentItem {
    id: string;
    colSpan: number;
    rowSpan: number;
    widget: AnyWidget;
}

interface PageTableData {
    rows: Record<string, unknown>[];
    totalElements: number;
    totalPages: number;
    currentPage: number;
    loading: boolean;
    appendLoading: boolean;
    hasMore: boolean;
    nextPage: number;
}
```

#### Props 전체 명세

```typescript
interface PageGridRendererProps {
    widgetItems: PageWidgetItem[];
    mode: RendererMode;   // 'preview' | 'live'

    // ── 빌더 미리보기 전용 ──────────────────────────────────────
    onItemClick?: (itemId: string) => void;
    selectedItemId?: string | null;

    // ── 공통 ────────────────────────────────────────────────────
    codeGroups?: CodeGroupDef[];
    urlParams?: Record<string, string>;
    currentSearchParams?: Record<string, string>;
    mainConnectedSlug?: string;
    leaveCheck?: boolean;

    // ── 검색 ────────────────────────────────────────────────────
    searchValues?: Record<string, string>;
    onSearchChange?: (fieldId: string, value: string) => void;
    onSearch?: (widgetId: string) => void;
    onReset?: (widgetId: string) => void;

    // ── 폼 ──────────────────────────────────────────────────────
    formValuesMap?: Record<string, Record<string, string>>;
    onFormValuesChange?: (widgetId: string, fieldId: string, value: string) => void;
    onContentAction?: (connectedContentWidgetIds: string[], action: 'save' | 'delete', goBackAfterAction?: boolean) => void;
    onDataSave?: (connectedContentWidgetIds: string[], dataSaveSlug: string, goBackAfterAction?: boolean, paramSave?: string) => void;

    // ── 탭 간 폼값 공유 ─────────────────────────────────────────
    crossTabFormValues?: Record<string, string>;
    onCrossTabFormChange?: (fieldId: string, value: string) => void;

    // ── 테이블 ──────────────────────────────────────────────────
    tableDataMap?: Record<string, PageTableData>;
    sortKeyMap?: Record<string, string | null>;
    sortDirMap?: Record<string, 'asc' | 'desc'>;
    onSort?: (widgetId: string, accessor: string, dir: 'asc' | 'desc' | null) => void;
    onPageChange?: (widgetId: string, page: number) => void;
    onLoadMore?: (widgetId: string) => void;
    tableSelectedRowsMap?: Record<string, number[]>;
    onTableRowsSelect?: (widgetId: string, ids: number[]) => void;

    // ── 카테고리 ────────────────────────────────────────────────
    categorySelections?: Record<string, number | null>;
    onCategorySelect?: (widgetId: string, selectedId: number | null) => void;

    // ── 서브리스트 ──────────────────────────────────────────────
    subListRowsMap?: Record<string, SubListRow[]>;
    onSubListRowsChange?: (widgetId: string, rows: SubListRow[]) => void;

    // ── 멀티셀렉트 ──────────────────────────────────────────────
    multiSelectValuesMap?: Record<string, number[]>;
    onMultiSelectChange?: (widgetId: string, ids: number[]) => void;
    multiSelectExtraFieldValuesMap?: Record<string, Record<number, Record<string, string>>>;
    onMultiSelectExtraFieldChange?: (widgetId: string, itemId: number, fieldKey: string, value: string) => void;
}
```

#### 사용 패턴

```tsx
// preview
<PageGridRenderer mode="preview" widgetItems={widgetItems} />

// live — useWidgetPageState의 gridProps spread
<PageGridRenderer
    mode="live"
    widgetItems={widgetItems}
    codeGroups={codeGroups}
    urlParams={urlParams}
    mainConnectedSlug={mainConnectedSlug}
    {...gridProps}   // useWidgetPageState 훅 반환값 전체 전달
/>
```

> PageGridRenderer 내부에서 자동으로 `GridCell → WidgetRenderer`를 조립한다.  
> 운영 페이지에서 GridCell을 직접 배치하는 코드를 작성할 필요가 없다.

### 4-4. WidgetRenderer

`widget.type`으로 **올바른 Renderer를 선택**한다.

```typescript
// 8가지 위젯 타입 — switch 분기
widget.type === 'search'      → SearchRenderer
widget.type === 'table'       → TableRenderer
widget.type === 'form'        → FormRenderer
widget.type === 'space'       → SpaceRenderer
widget.type === 'category'    → CategoryRenderer
widget.type === 'sublist'     → SubListRenderer
widget.type === 'multiselect' → MultiSelectRenderer
widget.type === 'tab'         → TabRenderer
```

**preview 모드:**
```tsx
<WidgetRenderer
    mode="preview"
    widget={anyWidget}
    contentColSpan={12}
/>
```

**live 모드 주요 props:**
```tsx
<WidgetRenderer
    mode="live"
    widget={anyWidget}
    contentColSpan={12}
    formValues={formValuesMap[wid] ?? {}}
    onFormValuesChange={(fieldId, value) => updateFormValue(wid, fieldId, value)}
    onContentAction={handleContentAction}
    subListRowsMap={subListRowsMap}
    onSubListRowsChange={(wid, rows) => setSubListRowsMap(prev => ({ ...prev, [wid]: rows }))}
    dataSlug="my-list"
    onRefresh={() => fetchData(0)}
/>
```

| live 전용 prop | 타입 | 용도 |
|---|---|---|
| `onContentAction` | `(widgetIds: string[], action: 'save'\|'delete', goBack?: boolean) => void` | Space 버튼 → Form·SubList 저장/삭제 |
| `onDataSave` | `(widgetIds: string[], slug: string, goBack?: boolean, paramSave?: string) => void` | datasave 타입 버튼 전용 |
| `subListRowsMap` | `Record<string, SubListRow[]>` | widgetId → SubList 행 데이터 |
| `onSubListRowsChange` | `(widgetId: string, rows: SubListRow[]) => void` | SubList 행 변경 |

**팝업 내부 SubList 처리 (WidgetRenderer 내부):**
```typescript
// 팝업이 열릴 때 초기화, 닫힐 때 소멸
const [popupSubListRowsMap, setPopupSubListRowsMap] = useState<Record<string, SubListRow[]>>({});

const handlePopupClose = () => {
    setPopupOpen(false);
    setPopupSubListRowsMap({});
};
```

### 4-5. RendererContainer ★ 핵심

**모든 Renderer가 공통으로 사용하는 최상위 컨테이너.** 직접 테두리·배경·grid를 구현하지 않는다.

```typescript
interface RendererContainerProps {
    children: React.ReactNode;
    showBorder?: boolean;      // 기본 true
    bgColor?: string;          // CSS 색상값 ('none' 시 투명)
    className?: string;
    contentColSpan?: number;   // 지정 시 내부 CSS Grid 활성화
}
```

```tsx
// 기본 (테두리·배경만, grid 없음)
<RendererContainer showBorder bgColor="none">
    {children}
</RendererContainer>

// grid 활성화 (Form, Space 등 필드 배치)
<RendererContainer contentColSpan={12} showBorder bgColor={bgColor}>
    <div style={{ gridColumn: 'span 6', gridRow: 'span 1' }}>필드A</div>
    <div style={{ gridColumn: 'span 6', gridRow: 'span 1' }}>필드B</div>
</RendererContainer>
```

**contentColSpan 전달 시 활성화되는 내부 grid:**
```typescript
{
    display: 'grid',
    gridTemplateColumns: `repeat(${contentColSpan}, 1fr)`,
    gridAutoRows: `${ROW_HEIGHT - GAP_SIZE}px`,   // 72px
    rowGap: `${GAP_SIZE}px`,                       // 8px
    columnGap: '8px',
    overflow: 'clip',
}
```

### 4-6. SubListRenderer — 동적 행 입력

Form 위젯과 함께 사용하는 **다건 행 입력 테이블** 렌더러.

```typescript
export interface SubListRow {
    _rowId: string;
    [key: string]: unknown;   // 컬럼 key → 값
}

interface SubListRendererProps {
    mode: RendererMode;
    widget: SubListWidget;
    rows?: SubListRow[];
    onChange?: (rows: SubListRow[]) => void;
    onFileChange?: (fieldId: string, files: File[], rowId?: string) => void;  // 파일 업로드 컬럼
}
```

```tsx
// preview — disabled 샘플 1행 자동 표시
<SubListRenderer mode="preview" widget={subListWidget} />

// live — 외부 상태로 제어
<SubListRenderer
    mode="live"
    widget={subListWidget}
    rows={subListRowsMap[widget.widgetId] ?? []}
    onChange={rows => onSubListRowsChange(widget.widgetId, rows)}
/>
```

| 모드 | 동작 |
|---|---|
| `preview` | disabled 샘플 행 1개. 추가/삭제 버튼 비활성 |
| `live` | + 버튼 → 빈 행 즉시 추가. 각 셀 입력 → 행 값 업데이트. 🗑️ → confirm 후 삭제 |

### 4-8. TabRenderer — 특수 구조

탭 전환 UI. **탭별 독립 상태 + lazy mount + keep-alive** 구조가 핵심.

```typescript
interface TabRendererProps {
    mode: RendererMode;
    widget: TabWidget;
    pageSlug?: string;
    parentMainConnectedSlug?: string;
    leaveCheck?: boolean;
}
```

```typescript
// types.ts
export interface TabItem {
    id: string;
    label: string;
    labelMsgKey?: string;
    pageSlug?: string;          // live 모드: 연결 페이지 슬러그
    items?: PageContentItem[];  // preview 모드: 인라인 위젯 배열
}

export interface TabWidget {
    type: 'tab';
    widgetId: string;
    tabs: TabItem[];   // 최대 5개
}
```

**모드별 동작:**

| 모드 | 렌더링 소스 | 마운트 방식 |
|---|---|---|
| `preview` | `TabItem.items` (인라인 위젯 배열) | 첫 번째 탭만 즉시 마운트 |
| `live` | `TabItem.pageSlug` → `fetchTemplateConfig` → DB 위젯 배열 | 첫 탭 클릭 시 마운트, 이후 `hidden` CSS로 keep-alive |

**lazy mount + keep-alive:**
```typescript
// 최초 클릭 시 한 번만 마운트, 이후 hidden으로 상태 유지
{mountedTabs.has(tab.id) && (
    <div hidden={activeTabId !== tab.id}>
        <LiveTabPanel pageSlug={tab.pageSlug} ... />
    </div>
)}
```

**각 LiveTabPanel은 독립적인 useWidgetPageState를 실행 — 탭 A/B 상태 완전 격리:**
```typescript
function LiveTabPanel({ pageSlug, ... }) {
    const widgetItems = useFetchedPage(pageSlug);
    const pageState = useWidgetPageState(widgetItems);
    return <PageGridRenderer mode="live" widgetItems={widgetItems} {...pageState.gridProps} />;
}
```

### 4-10. FieldRenderer — 날짜 필드 동작 규칙

**파일**: `make/_shared/components/renderer/FieldRenderer.tsx`

#### 날짜 필드 설정 키

```
date 필드:
  defaultDateOffset   — 오늘 기준 N일 전(양수) / N일 후(음수) 기본값 설정
  defaultDate         — 직접 날짜 지정 (offset=0일 때 활성화)
  disablePast         — true이면 기본값 날짜 이전 선택 비활성화

dateRange 필드:
  defaultStartDateOffset / defaultStartDate / disableStartPast  — 시작일
  defaultEndDateOffset   / defaultEndDate   / disableEndPast    — 종료일
```

#### min 계산 규칙 — 현재 value 기준 아님

```
date:
  disablePast = true  → min = 기본값 날짜 (또는 오늘)
  disablePast = false → min 없음

dateRange 시작일:
  disableStartPast = true  → min = 시작일 기본값 날짜
  disableStartPast = false → min 없음

dateRange 종료일:
  disableEndPast = true  → min = 종료일 기본값 날짜
  disableEndPast = false → min 없음 (시작일 값과 무관)
```

#### live 모드 기본값 초기화

```
widgetItems 로딩 완료 시 (useEffect, 1회 실행)
  → offset ≠ 0이면 오늘 기준 계산, offset = 0이면 defaultDate 사용
  → 이미 값이 있으면 덮어쓰지 않음
```

| 필드 타입 | 기본값 설정 키 |
|---|---|
| `date` | `defaultDateOffset` / `defaultDate` |
| `dateRange` | `defaultStart/EndDateOffset` / `defaultStart/EndDate` |
| `select` · `radio` · `checkbox` | `defaultOptionValue` |
| `input` · `hidden` 등 | `defaultValue` |

> **Form 신규 입력 모드**에서는 URL 파라미터(`?key=val`)가 기본값보다 우선 적용.

### 5-1. 진입점 A — builder-contents-layout (빌더 템플릿 레퍼런스)

```typescript
const TABS = [
```

### 5-2. 진입점 B — 빌더 미리보기 (make/*/page.tsx)

좌측 설정 패널에서 편집한 내용이 **실시간으로 반영**되는 미리보기 영역.

```
make/widget/page.tsx      — 자유 구성 빌더
make/list/page.tsx        — 코드 생성 빌더
make/grid-layout/page.tsx — 격자 레이아웃 빌더
```

```tsx
// 빌더 미리보기 공통 패턴
<div className="flex-1 ...">

        mode="preview"
        widgetItems={buildWidgetItems()}  // 좌측 패널 state → PageWidgetItem[]
        selectedItemId={editingItemId}    // 선택된 위젯 강조
```

> preview 모드에서는 `onContentAction`, `subListRowsMap` 등 live 전용 props를 전달하지 않는다.

### 5-3. 진입점 C — 운영 페이지 (widgetSub/[slug]/page.tsx)

실제 사용자가 보는 **live 모드** 페이지.  
**`useWidgetPageState` 훅**이 모든 런타임 상태를 통합 관리하고, `gridProps`를 spread해서 전달한다.

```
파일: bo/src/app/admin/widgetSub/[slug]/page.tsx
경로: /admin/widgetSub/{slug}
```

```tsx
// useWidgetPageState — 런타임 상태 통합 훅
const { gridProps, confirmLeave } = useWidgetPageState({
    pageSlug: slug,
    widgetItems,
    codeGroups,
    urlParams,
    mainConnectedSlug,
    // ...
});

// JSX
<PageLayout title={pageTitle} mode="live">
    <PageGridRenderer
        mode="live"
        widgetItems={widgetItems}
        codeGroups={codeGroups}
        urlParams={urlParams}
        mainConnectedSlug={mainConnectedSlug}
        {...gridProps}   // 검색/테이블/폼/카테고리/서브리스트/멀티셀렉트 상태 전체 포함
    />
</PageLayout>
```

**useWidgetPageState가 관리하는 상태:**
```typescript
// gridProps에 포함되어 전달되는 상태
searchValues, onSearchChange, onSearch, onReset
formValuesMap, onFormValuesChange
onContentAction, onDataSave
tableDataMap, sortKeyMap, sortDirMap, onSort, onPageChange, onLoadMore
tableSelectedRowsMap, onTableRowsSelect
categorySelections, onCategorySelect
subListRowsMap, onSubListRowsChange
multiSelectValuesMap, onMultiSelectChange
multiSelectExtraFieldValuesMap, onMultiSelectExtraFieldChange
crossTabFormValues, onCrossTabFormChange
```

> `mode="live"` — 격자선 없음 + 실제 데이터 바인딩 + 인터랙션 활성화.  
> 컴포넌트 스택은 빌더 미리보기와 완전히 동일.

```javascript
document.querySelectorAll('[style*="grid-auto-rows"]').forEach((rc, i) => {
```

## 9. 검증 체크리스트

- [ ] builder-contents-layout에 해당 위젯 탭이 있고 UI가 올바른가?
- [ ] 빌더 미리보기와 builder-contents-layout UI가 동일한가?
- [ ] 운영 페이지(live mode)와 빌더 미리보기 UI가 동일한가?
- [ ] 모든 Renderer가 `RendererContainer`를 사용하는가?
- [ ] `ROW_HEIGHT`, `GAP_SIZE` 상수를 import해서 사용하는가? (숫자 하드코딩 없는가?)
- [ ] RendererContainer의 `gridAutoRows`가 `ROW_HEIGHT - GAP_SIZE` (72px)인가?
- [ ] GridCell의 `overflow: clip`으로 인한 클리핑이 없는가? (DevTools 확인)
- [ ] 새 Renderer를 `WidgetRenderer.tsx` if 분기에 등록했는가?
- [ ] PageGridRenderer에 `widgetItems`(PageWidgetItem[])를 전달하는가? (`contents` prop 금지)
- [ ] live 모드에서 `onContentAction` 시그니처가 `(widgetIds: string[], action, goBack?) => void`인가?
- [ ] 운영 페이지에서 `useWidgetPageState` 훅을 사용하고 `{...gridProps}`를 spread하는가?
- [ ] SubList 위젯이 있다면 `subListRowsMap` / `onSubListRowsChange`를 전달하는가?
- [ ] MultiSelect 위젯이 있다면 `multiSelectValuesMap` / `onMultiSelectChange`를 전달하는가?
- [ ] Tab 위젯이 있다면 각 LiveTabPanel이 독립적인 `useWidgetPageState`를 사용하는가?
- [ ] Tab 전환 시 `hidden` CSS로 keep-alive 되는가? (unmount/remount 금지)
- [ ] layerpopup 모드에서 `layerType` 기준 `CenterPopupLayout` vs `RightDrawerLayout` 분기가 있는가?
- [ ] `date` 필드의 `disablePast = true` 시 min이 기본값 날짜로 고정되는가? (현재 value 기준 아님)
- [ ] `dateRange` 종료일 min이 시작일 값과 무관하게 `disableEndPast`만으로 결정되는가?
