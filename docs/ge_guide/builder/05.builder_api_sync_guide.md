> 작성일: 2026-07-11
> 대상: `/admin/system/api` 화면을 이해하거나, Slug Entity 파일생성으로 만든 신규 API를
>       시스템에 등록·연동하는 작업을 하는 모든 에이전트
> 관련 파일:

## 1. 개요

`api_info` 테이블은 프로젝트에 존재하는 `/api/v1/**` 엔드포인트를 관리자가 목록으로
조회·검색·활성화 여부 관리할 수 있게 해주는 메타 테이블이다.
[Slug Entity 파일생성](./04.builder_slug_entity_guide.md#10-파일생성코드-생성--slug-entity--be-java-파일-자동생성)으로
새 API를 만든 뒤, 이 화면에서 **동기화**해야 시스템에 등록되고 빌더에서 사용할 수 있게 된다.

## 2. 화면 기능

- 목록: 카테고리/메서드/**사용여부**/키워드로 검색, 페이징
- 사용여부(active): 목록에서 토글 스위치로 즉시 저장 가능
- 등록/수정/삭제: 수동으로 API 메타 정보 직접 관리 가능

## 3. API 동기화 — `@ApiLinkedEntity` 기반 자동 등록

**[API 동기화]** 버튼(`POST /api-infos/sync`) 클릭 시:

1. Spring MVC에 실제 등록된 모든 핸들러 매핑을 스캔
2. `/api/v1/`로 시작하는 URL 패턴만 대상으로 함
3. `api_info` 테이블에 **이미 존재하는 API는 건너뛰고(skipped)**, 없는 것만 신규 추가(added)
4. 신규 추가되는 API는 **`active=true`(사용)로 기본 등록**됨
5. 컨트롤러에 `@ApiLinkedEntity("EntityName")`이 붙어 있으면 `connectedEntity` 컬럼에 자동 기록

```java
@RestController
@RequestMapping("/api/v1/hero-data")
@ApiLinkedEntity("HeroData")   // ← Slug Entity 코드생성 시 자동으로 부여됨
public class HeroDataController { ... }
```

> 이미 등록된 API의 이름/설명 등을 수정해도 동기화는 덮어쓰지 않는다(추가 전용).
> 연결 엔티티만 갱신하려면 **[Entity 동기화]**(`POST /api-infos/sync-entities`, 기존 행의 `connected_entity`만 갱신, 신규 행 추가 없음)를 별도로 사용한다.

## 4. 활성 API 목록 — 빌더 연동용

인증된 사용자는 `GET /api-infos/active`로 **사용여부(active)=true인 API만** 페이징 없이
List로 받을 수 있다 (`SlugRegistry`/`SlugEntity`의 `/active` 엔드포인트와 동일 컨벤션).
이 목록은 [Space 빌더 버튼의 'API 연동' 연결](./00-1.builder_widget_components.md#6-2-space-위젯-conntype-목록)에서
autocomplete selectbox로 소비된다 — 미사용(active=false) API는 빌더에 노출되지 않는다.

## 5. 전체 흐름 요약

```
[Slug Entity 관리] 파일생성 (§04)
    │  6개 Java 파일 생성 + bo-api 재기동 (신규 테이블/컨트롤러 반영)
    ▼
[/admin/system/api] API 동기화
    │  신규 컨트롤러 자동 스캔 → api_info 테이블 등록 (active=true)
    ▼
[Space 빌더] 버튼 '연결' → 'API 연동' 선택
    │  GET /api-infos/active 목록에서 autocomplete로 선택 → apiInfoId 저장
    ▼
(선택) 사용 중지하려면 /admin/system/api 목록에서 사용여부 토글 OFF
```

## 6. Space 버튼 live 호출

Space 버튼의 'API 연동'은 `apiInfoId` 저장뿐 아니라, 버튼 클릭 시 실제로 해당 API를 호출하는 live 동작까지
구현·검증 완료됐다. 실행 로직 위치와 상세 흐름은 [`06.builder_entity_to_api_guide.md`](./06.builder_entity_to_api_guide.md) §8 참고
(`useWidgetPageState.ts`의 `handleApiCall`이 담당하며, live 호출 시에는 이 문서 §3의 `GET /api-infos/active`
목록을 클라이언트에서 조회해 매칭한다 — 관리자 전용인 단건 조회 API는 쓰지 않는다).
