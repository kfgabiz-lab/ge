# 00. 빌더 개요 (Builder Overview)

> 작성일: 2026-06-29  
> 대상: Bo 빌더 시스템을 처음 접하거나 전체 구조를 파악해야 하는 모든 에이전트

> - `bo/src/app/admin/templates/make/widget/page.tsx` — 빌더 진입점
> - `bo/src/app/admin/widgetSub/[slug]/page.tsx` — 운영 페이지 (빌더 결과물)
> - `bo/src/app/admin/templates/make/_shared/types.ts` — 위젯/필드 타입 정의

## 0. 한 문장 정의

**빌더는 운영자가 코드 없이 관리 화면을 조립하고, 그 결과를 `/admin/widgetSub/{slug}`에서 바로 운영하는 도구다.**

---

## 1. 왜 필요한가

화면마다 개발자가 직접 페이지를 코딩하면 두 가지 문제가 생긴다.

1. **속도** — 유사한 목록/등록 화면을 매번 새로 만들어야 함
2. **의존성** — 운영자가 화면을 바꾸려면 항상 개발자가 필요함

빌더는 이 두 문제를 해결한다.

- 운영자가 빌더에서 위젯을 조합해 화면을 직접 구성
- 저장 즉시 `/admin/widgetSub/{slug}` 경로로 운영 페이지 자동 생성
- 개발자는 새 위젯/필드 타입을 추가하는 역할만 담당

---

## 2. 무엇을 만들 수 있는가

### 2-1. 위젯 8종

빌더에서 사용할 수 있는 위젯 타입은 8가지다. 각 위젯은 화면의 한 블록을 담당한다.

| 위젯 타입 | 역할 | 주요 사용 화면 |
|---|---|---|
| `search` | 검색 조건 입력 폼 | 목록 화면 상단 |
| `table` | 데이터 목록 테이블 | 목록 화면 본문 |
| `form` | 데이터 입력/수정 폼 | 등록·수정 화면 |
| `space` | 버튼·텍스트 공간 영역 | 저장·취소 버튼 등 |
| `category` | 트리형 카테고리 선택 | 카테고리 관리 화면 |
| `sublist` | 다건 행 입력 테이블 | 상세 내역 입력 |
| `multiselect` | 다중 선택 위젯 | 복수 항목 선택 |
| `tab` | 탭 전환 레이아웃 | 탭별 독립 화면 구성 |

### 2-2. 만들 수 있는 화면 패턴

| 패턴 | 조합 예시 |
|---|---|
| 목록 화면 | search + table + space(버튼) |
| 등록·수정 화면 | form + space(저장/취소 버튼) |
| 카테고리 관리 | category + form + space |
| 상세 + 서브 내역 | form + sublist + space |
| 탭 기반 복합 화면 | tab (각 탭에 위젯 조합) |

## 3. 전체 흐름

```
[빌더] /admin/templates/make/widget
    운영자가 위젯 추가·크기 조정·필드 구성
         │
         ▼
    [저장] templateType = 'PAGE' 고정
    configJson = { widgetItems: [...] }
         │
         ▼
[운영 페이지] /admin/widgetSub/{slug}
    DB에서 configJson 로드
    → PageGridRenderer → GridCell → WidgetRenderer → FieldRenderer
    → 빌더에서 구성한 화면 그대로 렌더링
```

**핵심**: 어떤 위젯 조합이든 저장 타입은 `'PAGE'` 하나다. 운영 페이지는 templateType을 보지 않고 `widgetItems` 배열만 렌더링한다.

---

## 4. 핵심 용어 사전

| 용어 | 정의 |
|---|---|
| **위젯(Widget)** | 화면을 구성하는 최상위 블록. search·form·table 등 8종 |
| **컨텐츠(Content)** | 위젯 셀 안에 배치되는 단위. 하나의 위젯 셀에 복수 컨텐츠 배치 가능 |
| **필드(Field)** | Form·Search 위젯 안의 입력 요소. input·select·date 등 |
| **셀(Cell)** | 그리드 위에서 위젯이 차지하는 공간. colSpan × rowSpan으로 크기 결정 |
| **템플릿(Template)** | 빌더에서 저장한 위젯 구성 전체. DB의 `page_template` 테이블에 저장 |
| **slug** | 템플릿 고유 식별자 (영문/숫자/하이픈). 운영 페이지 URL에 사용 (`/widgetSub/{slug}`) |
| **configJson** | 템플릿의 위젯 구성 데이터. `{ widgetItems: [...] }` 형식 |
| **preview 모드** | 빌더 미리보기. 격자선 표시, 기능 비활성 |
| **live 모드** | 운영 페이지. 격자선 없음, 실제 데이터 바인딩·인터랙션 활성 |

## 6. 절대 금지 패턴

- `templateType === 'QUICK_LIST'` / `'QUICK_DETAIL'` 분기 — DB에 존재하지 않는 타입
- 운영 페이지(`widgetSub/[slug]`)에서 templateType으로 렌더링 분기
- configJson에 `widgetItems` 외 다른 최상위 키 추가

> 자세한 금지 패턴은 각 담당 문서(02~04)를 참조한다.
