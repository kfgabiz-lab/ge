# 01. 빌더 좌측 패널 (Input Left Panel) 가이드

> 작성일: 2026-05-03 / 최종 수정: 2026-06-29  
> 대상: 빌더 페이지 좌측 설정 패널을 새로 만들거나 수정하는 모든 에이전트  
> 선행 문서: `docs/ge_guide/builder/00.builder_overview.md` (빌더 전체 개요)

---

## 0. 핵심 원칙

빌더 페이지는 **좌측 설정 패널(Input)** + **우측 미리보기(Output)** 구조로 나뉜다.  
이 문서는 **좌측 설정 패널**만 다룬다. 우측 패널은 `02.input_builder_outputpanel.md` 참조.

```
<div className="grid grid-cols-[340px_1fr] gap-5 items-start">
  ┌──────────────────────┐   ┌──────────────────────────────┐
  │  좌측 Input (340px)   │   │   우측 Output (미리보기)       │
  │  ← 이 문서 대상        │   │   PageGridRenderer 담당       │
  └──────────────────────┘   └──────────────────────────────┘
```

---

## 1. 빌더 종류

| 빌더 | 경로 | templateType | 목적 | 위젯 구성 |
|---|---|---|---|---|
| `widget` | `make/widget` | `PAGE` | 자유 구성 — 8가지 위젯 타입 | 자유 추가/삭제/재정렬 |
| `list` | `make/list` | `LIST` | 검색+목록 페이지 **코드 생성** | 동적 필드 추가 |
| `grid-layout` | `make/grid-layout` | `PAGE` | 12칸 격자 레이아웃 목록 구성 | 동적 필드 추가 |

> **어느 빌더로 저장하든 `PAGE` 타입 결과물은 `/admin/widgetSub/{slug}`에서 동일하게 렌더링된다.**  
> `LIST` 타입은 코드 생성 전용 — 운영 페이지로 렌더링되지 않는다.

---

## 2. 좌측 패널 4-레이어 구조

모든 빌더 페이지의 좌측 패널은 아래 4-레이어 구조를 따른다.

```
<div className="bg-white border border-slate-200 rounded-xl sticky top-4">

  [Layer 1] <TemplateLoader>                  ← 불러오기 드롭다운
  ─────────────────────────────────────────
  [Layer 2] <OutputModePanel> (선택)           ← page/layerpopup 전환 필요 시만
  ─────────────────────────────────────────
  [Layer 3] 위젯 셀 영역 (스크롤)
            └── 위젯 헤더 (bg-slate-900)
            └── 컨텐츠 행 반복
                └── ContentRowHeader          ← 클릭 시 편집 토글
                └── SizeSettingPanel          ← 편집 중일 때만 표시
                └── CommonBuilderDispatcher   ← 편집 중일 때만 표시

</div>
```

**스크롤 영역 max-h 기준**:

| OutputModePanel 유무 | max-h 값 |
|---|---|
| 없음 (list, grid-layout 등) | `max-h-[calc(100vh-240px)]` |
| 있음 (widget, quick-detail) | `max-h-[calc(100vh-280px~320px)]` |

## 3. 핵심 컴포넌트

### 3-1. CommonBuilderDispatcher ★ 가장 중요

**모든 빌더 페이지에서 위젯 빌더를 렌더링할 때 반드시 이 컴포넌트를 경유해야 한다.**

```typescript
// ❌ 절대 금지 — 개별 빌더 직접 사용
<SearchWidgetBuilder widget={...} onChange={...} />
<SpaceBuilder widget={...} onChange={...} />

// ✅ 올바른 방법 — 항상 CommonBuilderDispatcher 경유
<CommonBuilderDispatcher
    widget={content.widget}
    onChange={w => setContent(prev => ({ ...prev, widget: w as WidgetType }))}
    context={{ ... }}
/>
```

**파일**: `make/_shared/components/builder/CommonBuilderDispatcher.tsx`

#### context Props 전체 명세

```typescript
context: {
    // Form·Table·Category의 DB Slug 선택 드롭다운에 사용
    slugOptions: { id: number; slug: string; name: string }[];

    // Space ActionButton 팝업 연결, Category 등록/수정 팝업에 사용
    pageTemplates?: TemplateItem[];

    // Table 위젯에서 연결할 Search 목록 (Table 있을 때 필수)
    searchWidgets?: Array<{ widgetId: string; contentKey: string }>;

    // Space ActionButton에서 연결할 Form 목록 (Form+Space 조합 시 필수)
    formWidgets?: Array<{ widgetId: string; contentKey: string; connectedSlug?: string }>;

    // Form 필드 colSpan 최대값 (기본 12, 우측 드로어 모드는 2)
    maxColSpan?: number;

    // Category 위젯에서 parentWidgetId 선택에 사용
    categoryWidgets?: { widgetId: string; label?: string; depth: number }[];

    // Space에서 ActionButton만 추가 가능하게 제한
    actionButtonOnly?: boolean;
}
```

#### 위젯 타입별 context 설정

| 위젯 타입 | 필수 context | 선택 context |
|---|---|---|
| `search` | — | `pageTemplates` |
| `table` | `slugOptions`, `searchWidgets` | `pageTemplates` |
| `form` | `slugOptions` | `pageTemplates`, `maxColSpan` |
| `space` | `pageTemplates` | `formWidgets`, `actionButtonOnly`, `maxColSpan` |
| `category` | `slugOptions`, `pageTemplates` | `categoryWidgets` |
| `sublist` | — | — |
| `multiselect` | — | — |
| `tab` | `pageTemplates` | — |

### 3-2. ContentRowHeader

컨텐츠 행 상단 헤더. 클릭 시 빌더 설정 패널 토글.

**파일**: `make/_shared/components/builder/ContentRowHeader.tsx`

```typescript
<ContentRowHeader
    widgetType="form"
    label={`Form${widget.contentKey ? ` — ${widget.contentKey}` : ''}`}
    colSpan={content.colSpan}
    rowSpan={content.rowSpan}
    isEditing={editingContentId === content.id}
    isFixed                                   // 고정 구조 시 (삭제 버튼 없음)
    onToggle={() => setEditingContentId(
        editingContentId === content.id ? null : content.id
    )}
    // 동적 구조에서만 사용:
    // onRemove={() => removeContent(itemId, contentId)}
    // dragHandleProps={handleProps}
/>
```

**위젯 타입별 헤더 색상**:

| widgetType | 색상 |
|---|---|
| `search` | 파란색 (blue-700) |
| `table` | 초록색 (emerald-700) |
| `form` | 보라색 (violet-700) |
| `space` | 황색 (amber-700) |
| `category` | 청록색 (cyan-700) |
| `sublist` | 남색 (indigo-700) |
| `multiselect` | 주황색 (orange-700) |
| `tab` | 분홍색 (pink-700) |

**라벨 작성 규칙**:
```typescript
// contentKey가 있으면 포함, 없으면 타입명만
label={`검색${widget.contentKey ? ` — ${widget.contentKey}` : ''}`}
label="공간영역"     // space는 contentKey 없음
label="데이터테이블"  // table도 contentKey 없음
```

### 3-3. SizeSettingPanel

colSpan/rowSpan 숫자 조정 패널. **ContentRowHeader 바로 아래, CommonBuilderDispatcher 바로 위**에 배치한다.

**파일**: `make/_shared/components/builder/SizeSettingPanel.tsx`

```typescript
<SizeSettingPanel
    colSpan={content.colSpan}
    rowSpan={content.rowSpan}
    maxColSpan={om.isRightDrawer ? 2 : 12}
    onColSpanChange={v => updateSize(v, content.rowSpan)}
    onRowSpanChange={v => updateSize(content.colSpan, v)}
/>
```

### 3-4. OutputModePanel

page / layerpopup 출력 모드 전환 탭. 팝업 타입/제목/크기 설정 포함.  
**widget, quick-detail에서 사용. list, grid-layout에서는 사용 안 함.**

**파일**: `make/_shared/components/builder/OutputModePanel.tsx`

```typescript
const om = useOutputMode();

<OutputModePanel
    outputMode={om.outputMode}
    layerType={om.layerType}
    layerTitle={om.layerTitle}
    layerWidth={om.layerWidth}
    onOutputModeChange={om.setOutputMode}
    onLayerTypeChange={om.setLayerType}
    onLayerTitleChange={om.setLayerTitle}
    onLayerWidthChange={om.setLayerWidth}
/>
```

**우측 드로어(`layerType='right'`) 전환 시 colSpan 클램핑 필수**:
```typescript
// Form + Space 양쪽 모두 colSpan 최대 2로 클램핑
useEffect(() => {
    if (om.outputMode === 'layerpopup' && om.layerType === 'right') {
        // Form 필드 클램핑
        setFormContent(prev => ({
            ...prev,
            colSpan: Math.min(prev.colSpan, 2),
            widget: { ...prev.widget, fields: prev.widget.fields.map(f => ({ ...f, colSpan: Math.min(f.colSpan, 2) })) },
        }));
        // Space 아이템 클램핑
        setSpaceContent(prev => ({
            ...prev,
            colSpan: Math.min(prev.colSpan, 2),
            widget: { ...prev.widget, items: prev.widget.items.map(i => ({ ...i, colSpan: Math.min(i.colSpan ?? 1, 2) as 1|2|3|4|5 })) },
        }));
    }
}, [om.layerType, om.outputMode]);
```

> **widget 빌더**는 `widgetItems` 배열 전체 순회하여 form/space 내부까지 일괄 클램핑. `make/widget/page.tsx` 참고.

### 3-5. useTemplateManagement ★ 불러오기·저장 통합 훅

**모든 빌더 페이지(widget, grid-layout 등)에서 공통 사용. templateType은 해당 빌더에 맞게 전달.**

**파일**: `make/_shared/hooks/useTemplateManagement.ts`

```typescript
// widget, grid-layout → PAGE
const tm = useTemplateManagement('PAGE');

// list → LIST
const tm = useTemplateManagement('LIST');
```

**반환값 전체**:

| 반환값 | 설명 |
|---|---|
| `filteredTemplates` | 검색어 기준 필터된 목록 |
| `isLoadingList` | 목록 로딩 중 여부 |
| `showLoadDropdown` / `setShowLoadDropdown` | 드롭다운 열림 상태 |
| `loadSearch` / `setLoadSearch` | 검색어 상태 |
| `isDeletingId` / `isDuplicatingId` | 삭제/복사 처리 중 ID |
| `loadTemplateList` | 목록 조회 함수 |
| `onLoadSuccess(tpl)` | 불러오기 성공 후 공통 처리 (ID 업데이트 + 드롭다운 닫기 + toast) |
| `handleDeleteTemplate(id)` | 삭제 처리 |
| `handleDuplicateTemplate(tpl)` | 복사 처리 |
| `currentTemplateId` / `currentTemplateName` | 현재 불러온 템플릿 정보 |
| `showSaveModal` / `setShowSaveModal` | 저장 모달 열림 상태 |
| `saveModalName` / `setSaveModalName` | 저장 모달 이름 |
| `saveModalSlug` / `setSaveModalSlug` | 저장 모달 slug |
| `saveModalDesc` / `setSaveModalDesc` | 저장 모달 설명 |
| `isSaving` | 저장 중 여부 |
| `openSaveModal()` | 저장 모달 열기 |
| `handleSaveConfirm(widgetItems, extra?)` | 저장 확인 (API 호출) |

**handleLoadSelect 패턴** — 반드시 마지막에 `tm.onLoadSuccess(tpl)` 호출:
```typescript
const handleLoadSelect = (tpl: TemplateItem) => {
    try {
        const config = JSON.parse(tpl.configJson);
        // 페이지별 고유 파싱
        setWidgetItems(config.widgetItems ?? []);
        setEditingItemId(null);
        tm.onLoadSuccess(tpl); // ← 반드시 마지막 (드롭다운 닫기 + toast)
    } catch {
        import('sonner').then(({ toast }) => toast.error('설정 파일 파싱에 실패했습니다.'));
    }
};
```

**저장 모달 연동 패턴**:
```typescript
<SaveModal
    show={tm.showSaveModal}
    onClose={() => tm.setShowSaveModal(false)}
    isEdit={!!tm.currentTemplateId}
    name={tm.saveModalName}   slug={tm.saveModalSlug}   desc={tm.saveModalDesc}
    isSaving={tm.isSaving}
    onNameChange={tm.setSaveModalName}
    onSlugChange={tm.setSaveModalSlug}
    onDescChange={tm.setSaveModalDesc}
    onConfirm={() => tm.handleSaveConfirm(buildWidgetItems(), { outputMode: om.outputMode })}
    toSlug={toSlug}
/>
```

### 3-6. TemplateLoader

불러오기 드롭다운 UI. **직접 드롭다운을 구현하지 않고 이 컴포넌트를 사용한다.**

**파일**: `make/_shared/components/builder/TemplateLoader.tsx`

```typescript
<TemplateLoader
    {...tm}   // isLoadingList, showLoadDropdown, loadSearch, filteredTemplates 등 일괄 전달
    onToggle={() => { tm.setShowLoadDropdown(v => !v); if (!tm.showLoadDropdown) tm.loadTemplateList(); }}
    onSearchChange={tm.setLoadSearch}
    onSelect={handleLoadSelect}          // 페이지별 고유 파싱 함수
    onDelete={tm.handleDeleteTemplate}
    onDuplicate={tm.handleDuplicateTemplate}
/>
```

### 3-7. useWidgetPageState ★ 런타임 페이지 상태 훅

> ⚠️ **빌더 훅이 아님** — 완성된 페이지(`/admin/widgetSub/{slug}`)의 런타임 데이터 처리 전용.  
> 빌더 페이지(make/widget, make/list 등)에서는 사용하지 않는다.

**파일**: `make/_shared/hooks/useWidgetPageState.ts`

#### useTemplateManagement vs useWidgetPageState 비교

| 항목 | useTemplateManagement | useWidgetPageState |
|---|---|---|
| 사용 위치 | 빌더 페이지 | 운영 페이지 |
| 목적 | 템플릿 저장/불러오기/삭제 | 폼/테이블/파일 런타임 데이터 관리 |
| 파일 | `hooks/useTemplateManagement.ts` | `hooks/useWidgetPageState.ts` |

#### 관리하는 상태 전체

```typescript
// 검색
searchValues: Record<string, string>

// 테이블
tableDataMap: Record<string, PageTableData>
sortKeyMap, sortDirMap: Record<string, string>
tableSelectedRowsMap: Record<string, number[]>

// 폼
formValuesMap: Record<string, Record<string, string>>

// 서브리스트
subListRowsMap: Record<string, SubListRow[]>

// 파일 업로드
fileValuesMap, existingFileMetaMap, imgBlobUrls
subListFileMap

// 멀티셀렉트
multiSelectValuesMap, multiSelectExtraFieldValuesMap

// 카테고리
categorySelections: Record<string, number | null>

// 수정 모드
currentGroupId: string | null
urlParamSaveExtras: Record<string, unknown>
```

#### 반환값 — gridProps

운영 페이지에서 `PageGridRenderer`에 전달하는 props 묶음:

```typescript
const { gridProps, confirmLeave } = useWidgetPageState({ pageSlug, ... });

<PageGridRenderer
    {...gridProps}
    widgetItems={configJson.widgetItems}
    mode="live"
    codeGroups={codeGroups}
/>
```

#### 특수 기능

| 기능 | 설명 |
|---|---|
| `enableUrlEditMode` | URL 파라미터(`?id`, `?group_id`) 기반 수정 모드 자동 진입 |
| `contentKey` 네임스페이스 | 탭 데이터 분리 — 각 탭의 폼 데이터가 겹치지 않게 관리 |
| `sharedDataId` | 탭 공유 row ID — 첫 탭 저장 후 나머지 탭이 수정 모드로 동작 |
| cross-form hideCondition | 다른 폼의 필드값을 조건으로 참조 가능 |
| `confirmLeave` | 폼 수정 후 페이지 이탈 시 확인 창 |

## 4. 빌더별 상세

### 4-1. widget 빌더

**경로**: `make/widget/page.tsx`  
**templateType**: `PAGE`  
**지원 위젯**: search, table, form, space, category, sublist, multiselect, tab (8가지)

#### 핵심 상태

```typescript
const [widgetItems, setWidgetItems] = useState<PageWidgetItem[]>([]);
const [editingItemId,    setEditingItemId]    = useState<string | null>(null);
const [editingContentId, setEditingContentId] = useState<string | null>(null);
```

#### widget 전용 기능

**i18n 모드** (BuilderI18nModeProvider):
```typescript
// 빌더 전체를 다국어 모드로 감쌈 — 필드 라벨 입력을 MessageKey 방식으로 전환
<BuilderI18nModeProvider>
    {/* 좌측 패널 + 우측 패널 */}
</BuilderI18nModeProvider>
```

**Slug Entity 자동 빌드**:
```typescript
// Entity 필드 기반으로 Form 위젯 필드 자동 생성
handleBuildFromEntity(entityFields, targetFormWidgetId);
```

**저장 전 유효성 검사**:
```typescript
validateBeforeSave() // contentKey/fieldKey 필수 여부 + 중복 검사
```

**위젯 추가 패턴**:
```typescript
// 새 위젯 아이템 추가
const newItem: PageWidgetItem = {
    id: generateId(),
    colSpan: 12,
    rowSpan: 6,
    contents: [createDefaultContent(widgetType)],
};
setWidgetItems(prev => [...prev, newItem]);
```

#### configJson 저장 구조

```json
{
    "widgetItems": [
        {
            "id": "wi-1",
            "colSpan": 12,
            "rowSpan": 6,
            "contents": [
                { "id": "ci-1", "colSpan": 12, "rowSpan": 5, "widget": { "type": "form", ... } },
                { "id": "ci-2", "colSpan": 12, "rowSpan": 1, "widget": { "type": "space", ... } }
            ]
        }
    ]
}
```

### 4-2. list 빌더

**경로**: `make/list/page.tsx`  
**templateType**: `LIST`  
**목적**: 검색폼 + 테이블 컬럼을 구성하여 완성된 List 페이지 TSX 코드를 생성

> ⚠️ `LIST` 타입은 운영 페이지(`/admin/widgetSub/{slug}`)로 렌더링되지 않는다. **코드 생성 전용.**

#### widget 빌더와의 차이점

| 항목 | widget | list |
|---|---|---|
| 저장 목적 | 운영 페이지 렌더링 | TSX 코드 파일 생성 |
| templateType | PAGE | LIST |
| 위젯 구성 | 자유 8종 | 검색 + 테이블 고정 (동적 필드 추가) |
| OutputModePanel | 있음 | 없음 |
| 코드 생성 | 없음 | `buildListTsxFile()` 있음 |

#### 핵심 기능 — 코드 생성

```typescript
// 검색 필드 + 테이블 컬럼 구성 완료 후 "코드 생성" 버튼 클릭
const generatedCode = buildListTsxFile({
    searchFields: [...],
    tableColumns: [...],
    buttonPosition: 'above' | 'between' | 'below',
    slug: templateSlug,
});
// GenerateModal에서 복사 가능한 TSX 코드 표시
```

#### context Props

```typescript
context={{
    slugOptions,
    pageTemplates,
    // actionButtonOnly: 버튼 영역에 필요한 경우 true
}}
```

### 4-3. grid-layout 빌더

**경로**: `make/grid-layout/page.tsx`  
**templateType**: `PAGE`  
**목적**: 12칸 격자 레이아웃 기반 검색+목록 페이지 구성 (코드 생성 없음)

#### 특징

- **DnD 지원**: `useSortableRows()` 훅으로 검색 필드/테이블 컬럼 드래그 순서 재정렬

- 저장 결과는 `/admin/widgetSub/{slug}`에서 렌더링

#### 사용 훅

```typescript
const tm = useTemplateManagement('PAGE');
const om = useOutputMode();
const { rows, addRow, removeRow, moveRow } = useSortableRows(...);
```

## 5. 컨텐츠 행 표준 패턴

```tsx
<div className="border-t border-slate-100">
    {/* 1. 헤더 — 항상 표시 */}
    <ContentRowHeader
        widgetType="form"
        label={`Form${widget.contentKey ? ` — ${widget.contentKey}` : ''}`}
        colSpan={content.colSpan}
        rowSpan={content.rowSpan}
        isEditing={editingContentId === content.id}
        onToggle={() => setEditingContentId(
            editingContentId === content.id ? null : content.id
        )}
    />

    {/* 2. 설정 영역 — 편집 중일 때만 표시 */}
    {editingContentId === content.id && (
        <div className="border-t border-slate-100 bg-slate-50/50">
            {/* 2-1. 크기 조정 */}
            <SizeSettingPanel
                colSpan={content.colSpan}
                rowSpan={content.rowSpan}
                maxColSpan={om.isRightDrawer ? 2 : 12}
                onColSpanChange={v => updateSize(v, content.rowSpan)}
                onRowSpanChange={v => updateSize(content.colSpan, v)}
            />
            {/* 2-2. 빌더 설정 */}
            <div className="px-3 pb-2 pt-1">
                <CommonBuilderDispatcher
                    widget={content.widget}
                    onChange={w => updateWidget(w as WidgetType)}
                    context={{ slugOptions, pageTemplates }}
                />
            </div>
        </div>
    )}
</div>
```

## 6. 위젯 타입별 context 실전 예시

```tsx
{/* Search */}
<CommonBuilderDispatcher widget={searchWidget} onChange={...}
    context={{ slugOptions, pageTemplates }} />

{/* Table — searchWidgets 필수 */}
<CommonBuilderDispatcher widget={tableWidget} onChange={...}
    context={{
        slugOptions, pageTemplates,
        searchWidgets: [{ widgetId: searchWidget.widgetId, contentKey: searchWidget.contentKey }],

    context={{
        slugOptions, pageTemplates,

    }} />

{/* Space — widget 빌더 (텍스트+버튼 자유) */}
<CommonBuilderDispatcher widget={spaceWidget} onChange={...}
    context={{
        slugOptions, pageTemplates,
        formWidgets: [{ widgetId: formWidget.widgetId, contentKey: formWidget.contentKey, connectedSlug: formWidget.connectedSlug }],

{/* Category */}
<CommonBuilderDispatcher widget={categoryWidget} onChange={...}
    context={{
        slugOptions, pageTemplates,

<CommonBuilderDispatcher widget={tabWidget} onChange={...}
    context={{ slugOptions, pageTemplates }} />
```

## 8. Category 위젯 — 카드 필드 키 설정 가이드

카테고리 카드에 표시할 필드(ID/CODE/TITLE/DESC)는 빌더에서 직접 키를 입력한다.  
연결된 Form이 `contentKey`를 사용해 중첩 저장하는 경우 `폼key.컬럼명` 형식으로 입력한다.

```
[저장된 dataJson 예시]
{
  "id": 553,
  "lv1pop": {          ← Form의 contentKey = 'lv1pop'
    "title": "대분류A",
    "code":  "A-001"
  }
}

[카드 필드 키]
TITLE → lv1pop.title   ← dataJson['lv1pop']['title']
CODE  → lv1pop.code    ← dataJson['lv1pop']['code']
ID    → lv1pop.id
```

| 입력 형식 | 읽기 경로 |
|---|---|
| `title` | `dataJson['title']` (root 직접) |
| `lv1pop.title` | `dataJson['lv1pop']['title']` (contentKey 중첩) |

### 금지 4. handleLoadSelect에서 onLoadSuccess 누락

```typescript
// ❌ 금지 — 드롭다운이 닫히지 않음
const handleLoadSelect = (tpl) => {
    setWidgetItems(JSON.parse(tpl.configJson).widgetItems);
    // onLoadSuccess 없음
};

// ✅ 올바른 방법
const handleLoadSelect = (tpl) => {
    setWidgetItems(JSON.parse(tpl.configJson).widgetItems ?? []);
    tm.onLoadSuccess(tpl); // ← 반드시 마지막에 호출
```

### 금지 5. 우측 드로어에서 colSpan 클램핑 누락

## 10. 검증 체크리스트

- [ ] `useTemplateManagement`를 사용하고 불러오기/저장 상태를 직접 관리하지 않는가?
- [ ] `TemplateLoader` 컴포넌트를 사용하고 드롭다운 UI를 직접 구현하지 않는가?
- [ ] `handleLoadSelect` 마지막에 `tm.onLoadSuccess(tpl)`을 호출하는가?
- [ ] `CommonBuilderDispatcher`만 사용하고 개별 빌더를 직접 사용하지 않는가?
- [ ] 모든 위젯에 `ContentRowHeader`가 있고 클릭 시 토글되는가?
- [ ] `ContentRowHeader` 아래에 `SizeSettingPanel`이 있는가?
- [ ] Table 위젯 context에 `searchWidgets`가 연결되어 있는가?
- [ ] Form + Space 조합에서 Space context에 `formWidgets`가 있는가?
- [ ] 우측 드로어 모드에서 Form context의 `maxColSpan`이 2인가?
- [ ] 우측 드로어 전환 시 Form + Space colSpan 클램핑 `useEffect`가 있는가?
- [ ] `onChange`에서 `as WidgetType` 타입 단언을 했는가?
- [ ] `max-h` 값이 OutputModePanel 유무에 맞게 설정되어 있는가?
- [ ] list 빌더라면 `useTemplateManagement('LIST')`를 사용하는가?
- [ ] 운영 페이지라면 `useWidgetPageState` 훅을 사용하는가?
- [ ] widget 빌더라면 `BuilderI18nModeProvider`로 감싸져 있는가?
